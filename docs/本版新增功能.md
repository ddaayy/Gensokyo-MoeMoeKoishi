# 本版新增功能

本 Fork（Te-River/Gensokyo-NewQQ）基于 [Hoshinonyaruko/Gensokyo](https://github.com/Hoshinonyaruko/Gensokyo) 修改，以下为所有扩展内容。

## 新增事件

| 事件 | 说明 | 对应 Intent |
|------|------|-------------|
| `GroupMessageEventHandler` | **非@群消息**接收。原版只支持 `GROUP_AT_MESSAGE_CREATE`（@消息），新增 `GROUP_MESSAGE_CREATE` 支持 | `GroupMessageEventHandler` |
| `FriendAddEventHandler` | 被用户添加为好友通知 | `FriendAddEventHandler` |
| `FriendDelEventHandler` | 被用户删除好友通知 | `FriendDelEventHandler` |
| `C2CMsgRejectHandler` | 用户关闭机器人 C2C 消息推送 | `C2CMsgRejectHandler` |
| `C2CMsgReceiveHandler` | 用户开启机器人 C2C 消息推送 | `C2CMsgReceiveHandler` |

## 新增 API

| Action | 说明 | 文件 |
|--------|------|------|
| `get_avatar` | 获取用户 QQ 头像直链（[文档](./额外api-get_avatar.md)） | get_avatar.go |
| `get_robot_share_link` | 获取机器人分享链接 | get_robot_share_link.go |
| `send_private_msg_wakeup` | 发送被动唤醒私聊消息 | send_private_msg_wakeup.go |
| `put_interaction` | 处理按钮交互回调 | put_interaction.go |

## 新增特性

### 群消息 `to_me` 字段

群消息事件新增 `to_me` 字段，标识消息是否 @ 了机器人：

```json
{
    "group_id": 123456,
    "message": [...],
    "to_me": true,
    "user_id": 789012
}
```

- `GROUP_AT_MESSAGE_CREATE` → `to_me: true` 始终为 true
- `GROUP_MESSAGE_CREATE` → 检测 `Mentions` 中是否有 `IsYou=true`

### 非@消息 @ 处理

`GROUP_MESSAGE_CREATE` 事件中的 @ 处理：

| 场景 | 行为 |
|------|------|
| 入站 `@bot` | 从 content 中剥离，`to_me = true` |
| 入站 `@其他人` | 转为 `[CQ:at,qq=虚拟ID]`，建立双向映射 |

### 出站 @ 格式化

| 场景 | 旧格式 | 新格式 |
|------|--------|--------|
| 纯文本出站 | `<@!OpenID>` | `<qqbot-at-user id="OpenID" />` |
| Markdown 出站 | 不支持 | `<qqbot-at-user id="OpenID" />`（自动转换 `[CQ:at]`） |

### Markdown 中的 @ 能力

在 Markdown 卡片内容中嵌入 `[CQ:at,qq=数字]`，Gensokyo 自动转换为 QQ API 的 `<qqbot-at-user>` 标签。

### 按钮虚拟 ID 自动转化

遍历 keyboard 中所有按钮的 `specify_user_ids`，将虚拟数字 ID 自动转为真实 QQ OpenID，确保 QQ API 能正确识别权限用户。

### C2C 按钮 permission 修复

C2C 单聊场景下 QQ 不支持 `permission.type=0`（指定用户），检测到 `specify_user_ids` 有值且 `type=0` 时强制改为 `type=2`（所有人）。

### action.type / permission.type 零值不被删除

修复 JSON `omitempty` 导致 `action.type=0`、`permission.type=0`、`render_data.style=0` 被序列化丢弃的问题。

### Sender.Nickname 自动填充

当 `card_nick` 配置为空时，自动从 `Author.Username` 填充 `event.sender.nickname`：

```json
{
    "sender": {
        "nickname": "用户名",   // 从 Author.Username 获取
        "user_id": 123456789
    }
}
```

优先顺序：`card_nick` 配置值 > `Author.Username` > 空

### event_id 全场景存储

| 事件 | event_id 来源 |
|------|---------------|
| `INTERACTION_CREATE` | ProcessInlineSearch |
| `C2C_MSG_RECEIVE` | process_c2c_switch |
| `FRIEND_ADD` | process_friend |
| `GROUP_ADD_ROBOT` | ProcessGroupAddBot |
| `GROUP_MSG_RECEIVE` | ProcessGroupMsgReceive |

### 静态编译

CI 构建使用 `CGO_ENABLED=0`，兼容旧版 GLIBC。

## 修复

- 按钮 `action.type`, `permission.type`, `render_data.style` 设置为 0 时被 JSON omitempty 删除
- 按钮中虚拟 ID 自动转化在私聊中失效（C2C permission.type 强制设为 2）
- `GetRemoveBotAtGroup` 误删用户主动发送的纯 @ 消息
- 入站 @bot 剥离逻辑使用 `Mentions[i].OpenID` 而非 `BotID`
- `ParseAndHandle` 添加错误日志与 panic 恢复
