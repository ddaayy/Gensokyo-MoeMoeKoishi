# 本版新增功能

本 Fork（Te-River/Gensokyo-NewQQ）基于 [Hoshinonyaruko/Gensokyo](https://github.com/Hoshinonyaruko/Gensokyo) 修改，以下为所有扩展内容。

## 新增事件

| 事件 | 说明 | 对应 Intent | 适用范围 |
|------|------|-------------|:--------:|
| `GroupMessageEventHandler` | **非@群消息**接收。原版只支持 `GROUP_AT_MESSAGE_CREATE`（@消息），新增 `GROUP_MESSAGE_CREATE` 支持 | `GroupMessageEventHandler` | 🏷️ 群聊 |
| `MemberEventHandler` | **频道成员变动**通知。`GUILD_MEMBER_ADD` → `notice.group_increase`（`sub_type=member`），`GUILD_MEMBER_REMOVE` → `notice.group_decrease`（`sub_type=member`） | `MemberEventHandler` | 📡 频道 |
| `GroupMemberAddEventHandler` | **群成员新增**通知（QQ API 未文档化事件）。→ `notice.group_increase`（`sub_type=member`） | `GroupMemberAddEventHandler` | 🏷️ 群聊 |
| `GroupMemberRemoveEventHandler` | **群成员移除**通知（QQ API 未文档化事件）。→ `notice.group_decrease`（`sub_type=member`） | `GroupMemberRemoveEventHandler` | 🏷️ 群聊 |

## 新增特性

### 群消息 `to_me` 字段

> **适用范围**：🏷️ 群聊

群消息事件新增 `to_me` 字段，标识消息是否 @ 了机器人。

```json
{
    "group_id": 123456,
    "message": [...],
    "to_me": true,
    "user_id": 789012
}
```

- `GROUP_AT_MESSAGE_CREATE` → `to_me: true` 始终为 true
- `GROUP_MESSAGE_CREATE` → 检测 `Mentions` 中是否有 `IsYou=true`

### 非@消息 @ 处理

> **适用范围**：🏷️ 群聊（仅 `GROUP_MESSAGE_CREATE`）

`GROUP_MESSAGE_CREATE` 事件中的 @ 处理。

| 场景 | 行为 |
|------|------|
| 入站 `@bot` | 从 content 中剥离，`to_me = true` |
| 入站 `@其他人` | 转为 `[CQ:at,qq=虚拟ID]`，建立双向映射 |

### 出站 @ 格式

> **适用范围**：🏷️ 群聊 + 📡 频道

| 场景 | 旧格式 | 新格式 |
|------|--------|--------|
| 纯文本出站 | `<@!OpenID>` | `<qqbot-at-user id="OpenID" />` |
| Markdown 出站 | 不支持 | `<qqbot-at-user id="OpenID" />`（自动转换 `[CQ:at]`）|

### Markdown 中的 @ 能力

> **适用范围**：🏷️ 群聊 + 📡 频道

在 Markdown 卡片内容中嵌入 `[CQ:at,qq=数字]`，Gensokyo 自动转换为 QQ API 的 `<qqbot-at-user>` 标签。

### Markdown 图片自动上传

> **适用范围**：🏷️ 群聊 + 📡 频道

Markdown 内容中的 `![](path)` 图片，HTTP(S) URL 直接保留，本地文件路径自动读取并上传到 QQ CDN 替换为直链。

### 按钮虚拟 ID 自动转化

> **适用范围**：🌐 全场景

遍历 keyboard 中所有按钮的 `specify_user_ids`，将虚拟数字 ID 自动转为真实 QQ OpenID，确保 QQ API 能正确识别权限用户。

### C2C 按钮 permission 修复

> **适用范围**：💬 C2C 私聊

C2C 单聊场景中，QQ 不支持 `permission.type=0`（指定用户），检测到 `specify_user_ids` 有值且 `type=0` 时强制改为 `type=2`（所有人）。

### action.type / permission.type 零值不被删除

> **适用范围**：🌐 全场景

修复 JSON `omitempty` 导致 `action.type=0`、`permission.type=0`、`render_data.style=0` 被序列化丢弃的问题。

### 语音编码优化（免 ffmpeg）

> **适用范围**：🌐 全场景（群聊 + 频道 + C2C 均支持语音发送）

支持 `[CQ:record]` 语音消息的自动转码（编码为 SILK v3），无需安装 ffmpeg。

| 格式 | 解码方式 | 说明 |
|------|----------|------|
| **WAV** (RIFF/WAVE) | 直接提取 PCM | 16-bit 单声道、匹配采样率时直接提取 |
| **MP3** | Go 解码 | 使用 `go-mp3` 库，无外部依赖 |
| 其他 (OGG/FLAC/AAC 等) | 回退 ffmpeg | 安装 ffmpeg 后可支持 |

> 语音文件必须符合 **16-bit、单声道、采样率 24000**（可在配置中调整 `record_sampleRate`）。不满足时会自动尝试通过 ffmpeg 转换。

### [CQ:reply] → message_reference 引用回复

`[CQ:reply,id=数字]` 不再被丢弃或作为纯文本发送，改为：

- 自动从文本中剥离
- 若同时存在 MD 卡片，将 `message_reference` 挂载到 MD 消息上合并发送

> ⚠️ **重要限制：**
> - 引用回复**仅在频道中**可正常显示引用样式（`GROUP_AT_MESSAGE_CREATE`）
> - **群聊中**（`GROUP_MESSAGE_CREATE`）QQ API **接受但不会渲染**引用样式——API 返回 200，但用户看不到引用效果
> - **Markdown 消息** QQ API **不支持** `message_reference` 参数，引用字段会被静默忽略

**`message_reference` 字段结构：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `message_id` | string | 被引用的消息 ID（自动从 `[CQ:reply,id=...]` 解析） |
| `ignore_get_message_error` | bool | 忽略获取消息时的错误，默认为 `true` |

### idmap MultiMap 身份图 + 一次性迁移 + msg_id 1h TTL

> **适用范围**：🌐 全场景

将旧版单值 `idmap.db` 改为显式类型的身份图，解决一个 vUIN 无法绑定多个 OpenID / rUIN 的问题：

| 文件 | 桶 | 用途 |
|------|-----|------|
| `openid-map.db` | `identity_to_vuin` | OpenID/rUIN/raw 身份 → 内部 vUIN |
| `openid-map.db` | `vuin_to_identity` | vUIN → 多个外部身份 |
| `openid-map.db` | `last_seen` | 当前消息上下文反查 |
| `openid-map.db` | `config` / `UserInfo` | 运行时配置与用户信息缓存 |
| `msgid-map.db` | `msg_to_virtual` / `virtual_to_msg` / `expires_at` | message_id 映射与过期时间 |
| `idmap.db`（旧） | — | 旧版单库，只作为一次性迁移源 |

**身份规则：**

- 下游默认继续使用内部 vUIN，不把黑盒 OpenID 直接暴露为 OneBot 数字 ID。
- OpenID 以 `openid:QQ:<appid>:<openid>` 记录。
- rUIN 以 `rUIN-<PLATFORM>-<index>-<id>` 对外表示，内部规范化为 `ruin:<PLATFORM>:<index>:<id>`。
- 多个 OpenID、多个 rUIN 可以绑定到同一个 vUIN。
- 反查 vUIN 时只选择当前消息上下文中的 OpenID/rUIN，不全量附加所有身份。

**新增/调整配置：**

```yaml
op_userid_type: vuin     # vuin/raw/ruin，下游 user_id/group_id 默认使用内部 vUIN
msgid_ttl_seconds: 3600  # message_id 映射默认保留 1 小时
```

`idmap_pro` 不再需要，MultiMap 身份图原生支持多身份绑定。

**迁移流程：**

1. 启动时检测到旧 `idmap.db`，阻塞转换到 `openid-map.db` / `msgid-map.db`。
2. 保留旧 `ids` 中已有 OpenID/raw 与 vUIN 的关系。
3. 拆解旧 `idmap_pro` 复合键，转成普通身份关系。
4. 复制 `config` 与 `UserInfo`。
5. 旧 `cache` 不全量迁移，只同步 message_id counter 避免撞号。
6. 校验后写入迁移标记；后续热路径不再访问旧库。

**msg_id 缓存：**

- 每次写入 message_id 时记录真实 ID、虚拟 ID 与过期时间。
- 默认 TTL 为 **1 小时**，后台每分钟清理过期条目。
- 旧 `cache` 桶不做全量迁移，避免搬入大量历史临时数据。

### Sender.Nickname 自动填充

自动使用 `Author.Username` 填充 `event.sender.nickname`：

```json
{
    "sender": {
        "nickname": "用户名",   // 从 Author.Username 获取
        "user_id": 123456789
    }
}
```

优先顺序：`card_nick` 配置 > `Author.Username` > 空

### 群成员变动通知

> **适用范围**：🏷️ 群聊

`GROUP_MEMBER_ADD` / `GROUP_MEMBER_REMOVE` 事件支持（独立 Intent `1<<24`，需在 `text_intent` 中启用 `GroupMemberAddEventHandler` 和 `GroupMemberRemoveEventHandler`）。

Gensokyo 捕获事件后以后端推送标准 OneBot V11 通知事件，`message` 字段附带 CQ 码：

```json
{"post_type":"notice","notice_type":"group_increase","sub_type":"approve","message":"[CQ:member,type=add,group_id=821404315,user_id=3607918353]",...}
```

#### 双向流程

| 阶段 | 方向 | 说明 |
|:----:|:----:|------|
| Gsk → 后端 | 入站 | `notice.group_increase.approve` / `notice.group_decrease.leave`，`message` 字段含 `[CQ:member,...]` |
| 后端 → Gsk | 出站 | 后端用普通 `send_group_msg` 回复，消息中包含 `[CQ:member]` + `[CQ:at]` + `[CQ:markdown]`。Gsk 自动解析 CQ 码、转换虚拟 ID、合并 @ 到 markdown |

详见 [cq-member.md](./cq码/扩展CQ码/扩展cq码-cq-member.md) 文档。

#### nonebot2 示例

```python
from nonebot.adapters.onebot.v11 import GroupIncreaseNoticeEvent, GroupDecreaseNoticeEvent

@on_notice().handle()
async def handle_group_increase(bot: Bot, event: GroupIncreaseNoticeEvent):
    # 读取 message 字段中的 CQ 码
    cq_msg = event.message  # "[CQ:member,type=add,user_id=3607918353]"
    await bot.send_group_msg(
        group_id=event.group_id,
        message=f"{cq_msg}欢迎新成员！"
    )

@on_notice().handle()
async def handle_group_decrease(bot: Bot, event: GroupDecreaseNoticeEvent):
    cq_msg = event.message
    await bot.send_group_msg(
        group_id=event.group_id,
        message=f"{cq_msg}离开了我们"
    )
```

### 频道成员变动通知

> **适用范围**：📡 频道

`GUILD_MEMBERS (1 << 1)` Intent 支持，接收频道成员加入/离开事件并转为 OneBot 标准 notice。

| QQ API 事件 | OneBot 通知类型 | sub_type |
|:-----------:|:---------------:|:--------:|
| `GUILD_MEMBER_ADD` | `notice.group_increase` | `member` |
| `GUILD_MEMBER_REMOVE` | `notice.group_decrease` | `member` |

自动建立新成员的 OpenID 与虚拟 ID 映射。需在 `text_intent` 中取消 `MemberEventHandler` 注释启用。

### 群消息 `sender.role` 支持 `member_role`

> **适用范围**：🏷️ 群聊

群消息 `sender` 中的 `role` 字段现在会优先使用 QQ API 返回的真实身份标识，而非仅依赖配置文件中的 `master_id`。

#### 字段映射

| `member_role` 值 | `sender.role` 输出 | 含义 |
|:-----------------:|:------------------:|:----:|
| `owner` | `owner` | 群主 |
| `admin` | `admin` | 管理员 |
| `member` | `member` | 普通群员 |

#### 上报示例

群主发消息时，`sender` 如下：

```json
{
    "sender": {
        "user_id": 10000001,
        "role": "owner",
        "nickname": "群主"
    }
}
```

#### 优先级
1. QQ API 返回的 `member_role`（`owner` / `admin` / `member`）→ **优先使用**
2. 配置 `master_id` 列表匹配 → 设为 `owner`
3. 以上均不满足 → 设为 `member`

#### nonebot2 获取方式

```python
from nonebot.adapters.onebot.v11 import GroupMessageEvent

@matcher.handle()
async def _(event: GroupMessageEvent):
    role = event.sender.role  # "owner" / "admin" / "member"
    if role == "owner":
        await matcher.finish("群主你好")
```

## 修复

- 按钮 `action.type`, `permission.type`, `render_data.style` 设置为 0 时被 JSON omitempty 删除
- 按钮中虚拟 ID 自动转化在私聊中失效（C2C permission.type 强制设为 2）
- `GetRemoveBotAtGroup` 误删用户主动发送的 @ 消息
- 入站 @bot 剥离逻辑使用 `Mentions[i].OpenID` 而非 `BotID`
- `ParseAndHandle` 添加错误日志和 panic 恢复
